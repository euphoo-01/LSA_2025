3. Токены называются лексеммами
4. Существующие заголовочные файлы должны оставаться в том виде, что есть. Их можно редактировать только с моего разрешения.
5. Если действие меняет архитектуру проекта, нужно запросить подтверждение у меня.
6. Заголовочные файлы хранятся в ./Headers/
7. Файлы реализации хранятся в ./Source/
8. Код обязательно должен быть разделен на различные модули. Модули должны разделять объявления в заголовочных файлах и реализацию в файлах реализации

### Общие правила языка LSA-2025

6.  **Основные Характеристики:**
    *   **Кодировка:** Все исходные файлы должны использовать кодировку **Windows-1251**.
    *   **Типы данных:** Поддерживаются три типа: `unsigned integer`, `char`, `logic`. Пользовательские типы не допускаются.
    *   **Преобразование типов:** Разрешено только автоматическое приведение типа `char` к `unsigned integer`. Любые другие преобразования запрещены.

7.  **Идентификаторы:**
    *   **Длина:** Максимальная длина идентификатора — **16 символов**.
    *   **Формат:** Должен начинаться с буквы (латинской или русской), за которой могут следовать буквы или цифры.
    *   **Ограничения:** Нельзя использовать в качестве идентификаторов ключевые слова (`unsigned`, `integer`, `char`, `logic`, `main`, `send`, `if`, `differ`, `because`, `func`).

8.  **Литералы:**
    *   **Целочисленные:** Могут быть десятичными (диапазон `0`–`4294967295`) или шестнадцатеричными (с префиксом `0x`).
    *   **Символьные:** Заключаются в одинарные кавычки (например, `'A'`). Поддерживается спецсимвол `\n`.

9.  **Синтаксис и Структура:**
    *   **Объявление:** Строго по формату `<тип> <идентификатор>;`. Только одна переменная на одно объявление.
    *   **Инициализация:** `<тип> <идентификатор> = <значение>;`.
    *   **Оператор деления:** Используется символ двоеточия (`:`).
    *   **Завершение инструкций:** Каждая инструкция должна заканчиваться точкой с запятой (`;`).
    *   **Выражения:** Должны быть записаны в одну строку.

10. **Структура Программы:**
    *   **Точка входа:** Функция `main` обязательна, должна быть единственной и возвращать значение типа `unsigned integer` через оператор `send`.
    *   **Объявление функций:** Все пользовательские функции (`func`) должны быть объявлены **до** точки входа `main`.
    *   **Область видимости:** Идентификаторы функций имеют глобальную область видимости, остальные идентификаторы — блочную.

11. **Семантические Правила:**
    *   Запрещено использование необъявленных переменных.
    *   Запрещено повторное объявление переменной в одной и той же области видимости.
    *   Типы данных при присваивании и в параметрах функций должны строго совпадать (с учётом разрешённого приведения `char` к `unsigned integer`).
    *   Деление на ноль является ошибкой.

12. **Стандартная Библиотека:**
    *   Функции `sqrt`, `pow`, `isPrime`, `toUpper`, `getMin`, `getMax`, `readch`, `writech` доступны для использования по умолчанию и не требуют подключения.

13. **Обработка Ошибок:**
    *   Трансляция прекращается после обнаружения **первой же ошибки**.
    *   Коды ошибок должны соответствовать установленным диапазонам:
        *   **Лексический анализ:** `111-125`
        *   **Синтаксический анализ:** `600-610`
        *   **Семантический анализ:** `126-136`
